# Capacity Modeling

Capacity modeling matches **maintenance demand** against **available staff** to answer three questions:

1. **Do we have enough staff?** — Is demand within capacity for each shift?
2. **When are we over or under?** — Which days and shifts need attention?
3. **What's the trend?** — Is demand growing, shrinking, or seasonal?

## How It Works

Work packages create **demand** (measured in man-hours). Shifts and headcount define **capacity** (productive hours available). The system computes **utilization** — the ratio of demand to capacity — for every day and shift, then visualizes it as a heatmap, chart, and table.

Seven **lenses** let you view the same data from different angles: planned demand, customer allocations, flight events, forecasted demand, actual hours worked, billed hours, and concurrent aircraft pressure.

## Who Uses It

- **Operations managers** view the `/capacity` page to monitor utilization, spot overloaded shifts, and plan staffing
- **Administrators** configure the model via `/admin/capacity` — setting headcounts, allocations, assumptions, and entering operational data
