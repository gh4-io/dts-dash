# Capacity API Reference

All capacity endpoints require authentication. Admin endpoints (`/api/admin/capacity/*`) require `admin` or `superadmin` role. The overview endpoint (`/api/capacity/overview`) is available to any authenticated user.

## Overview Endpoint

### GET /api/capacity/overview

Returns the complete capacity analysis for a date range, including demand, capacity, utilization, and all overlay data (allocations, events, forecast, worked, billed, concurrency).

**Auth:** Any authenticated user

#### Query Parameters

| Param | Type | Default | Description |
|-------|------|---------|-------------|
| `start` | ISO date | 7 days ago | Start of date range |
| `end` | ISO date | 23 days ahead | End of date range |
| `operators` | comma-separated | *(all)* | Filter by customer names |
| `aircraft` | comma-separated | *(all)* | Filter by aircraft registrations |
| `types` | comma-separated | *(all)* | Filter by aircraft types |

#### Example Request

```bash
curl -H "Authorization: Bearer <token>" \
  "http://localhost:3000/api/capacity/overview?start=2026-02-01&end=2026-02-28"
```

## Admin CRUD Endpoints

All admin endpoints follow the same pattern:

| Method | Pattern | Description |
|--------|---------|-------------|
| GET | `/api/admin/capacity/{feature}` | List all records (optional date range filter) |
| POST | `/api/admin/capacity/{feature}` | Create a new record |
| GET | `/api/admin/capacity/{feature}/[id]` | Get a single record |
| PUT | `/api/admin/capacity/{feature}/[id]` | Update a record |
| DELETE | `/api/admin/capacity/{feature}/[id]` | Delete a record |

**Auth:** All require `admin` or `superadmin` role.

**Error responses:** 400 (validation), 403 (forbidden), 404 (not found), 409 (conflict), 500 (server error).

### Demand Allocations

**Base path:** `/api/admin/capacity/demand-allocations`

**List:** `GET ?start=YYYY-MM-DD&end=YYYY-MM-DD` (both optional)

**Create/Update fields:**

| Field | Type | Required | Notes |
|-------|------|----------|-------|
| `customerId` | number | Yes | Customer ID |
| `shiftId` | number | No | Null = all shifts |
| `dayOfWeek` | number | No | 0=Sun..6=Sat, null = all days |
| `effectiveFrom` | YYYY-MM-DD | Yes | Start date |
| `effectiveTo` | YYYY-MM-DD | No | End date (null = indefinite) |
| `allocatedMh` | number | Yes | Must be > 0 |
| `mode` | string | Yes | `ADDITIVE` or `MINIMUM_FLOOR` |
| `reason` | string | No | Description |
| `isActive` | boolean | No | Default: true |

### Forecast Models

**Base path:** `/api/admin/capacity/forecast-models`

**Create/Update fields:**

| Field | Type | Required | Notes |
|-------|------|----------|-------|
| `name` | string | Yes | Model name |
| `description` | string | No | |
| `method` | string | Yes | `moving_average`, `weighted_average`, `linear_trend` |

```json
}
```

### Staffing Configurations

**Base path:** `/api/admin/capacity/staffing-configs`

**Create/Update fields:**

| Field | Type | Required | Notes |
|-------|------|----------|-------|
| `name` | string | Yes | Configuration name |
| `description` | string | No | |

**Additional endpoints:**

| Method | Path | Description |
|--------|------|-------------|
| POST | `[id]/activate` | Set as active configuration (deactivates others) |

### Staffing Shifts

**Base path:** `/api/admin/capacity/staffing-shifts`

**List:** `GET ?configId=<required>`

**Create/Update fields:**

| Field | Type | Required | Notes |
|-------|------|----------|-------|
| `configId` | number | Yes | Parent configuration |
| `name` | string | Yes | Shift line name |
| `category` | string | Yes | `DAY`, `SWING`, `NIGHT`, `OTHER` |
| `rotationId` | number | Yes | Rotation pattern to follow |
| `rotationStartDate` | YYYY-MM-DD | Yes | Cycle start date |
| `startHour` | number | Yes | 0-23 |
| `endHour` | number | Yes | 0-23 |
| `headcount` | number | Yes | Non-negative |
| `startMinute` | number | No | 0-59 |
| `endMinute` | number | No | 0-59 |
| `breakMinutes` | number | No | Break duration |
| `lunchMinutes` | number | No | Lunch duration |
| `mhOverride` | number | No | Override computed MH |
| `isActive` | boolean | No | Default: true |
| `sortOrder` | number | No | Display order |

**Bulk endpoint:** `POST /api/admin/capacity/staffing-shifts/bulk`

```json
{
  "action": "activate | deactivate | delete | update-category",
  "ids": [1, 2, 3],
  "category": "DAY"
}
```
