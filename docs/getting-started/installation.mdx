# Installation

## Prerequisites

| Requirement | Version |
|-------------|---------|
| Node.js | 20.x or later |
| npm | 10.x or later |
| OS | Windows (WSL2), macOS, or Linux |

## Steps

### 1. Clone the repository

```bash
git clone <repo-url> dashboard
cd dashboard
```

### 2. Install dependencies

```bash
npm install
```

### 3. Set up environment

Copy the pre-filled dev template:

```bash
cp docker/.env.dev.example .env.local
cp server.config.dev.yml server.config.yml
```

Or create `.env.local` manually with just the required secret:

```bash
npm run generate-secret
# Paste the output below
```

```env
AUTH_SECRET=<paste-generated-secret-here>
```

> See [Configuration](/docs/getting-started/configuration) for all available variables.

### 4. Start the development server

```bash
npm run dev
```

Open [http://localhost:3000](http://localhost:3000).

The database **auto-bootstraps on first start** — tables, migrations, system user, and default config are created automatically. No manual seed step is required.

### 5. Create your first user

Navigate to [http://localhost:3000/register](http://localhost:3000/register). The first user is automatically granted the `superadmin` role.

If you used `docker/.env.dev.example`, you can also run `npm run db:seed` to create the pre-configured admin (`admin@local` / `admin123`).

### 6. Seed reference data (recommended)

```bash
npm run db:seed-reference
```

Populates aircraft type mapping rules, manufacturers, aircraft models, and engine types. Idempotent — safe to re-run after any reset.

### 7. Add work package data

Place your SharePoint OData JSON export at `data/input.json`. The file should be either:
- A bare JSON array: `[{...}, {...}]`
- OData format: `{"value": [{...}, {...}]}`

## Troubleshooting

### Build fails with stale cache

```bash
rm -rf .next
npm run build
```

### WSL: SWC binary lock

```bash
rm -rf node_modules/@next/swc*
npm install
```
